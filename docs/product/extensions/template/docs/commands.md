# Commands

> Replace this template with the actual CLI reference of the extension.

For each subcommand, document:

- **Name.** `python -m memory ext <id> <subcommand>`.
- **Summary.** One line.
- **Synopsis.** Full argument syntax, including required and optional flags.
- **Description.** What it does, when to use it, side effects.
- **Examples.** At least two: a typical case and an edge case.
- **Exit codes.** When non-zero, what each code means.
- **See also.** Related subcommands or Mirror commands.

Group subcommands by purpose, not by alphabet (CRUD, imports, reports,
migration, etc.). Put the most-used ones first.

Keep examples copy-pasteable. Use realistic but anonymized data.
